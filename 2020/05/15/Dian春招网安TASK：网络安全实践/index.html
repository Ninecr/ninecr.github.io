很有趣的几道CTF入门向的题，花了差不多两天的时间，张三进度太快了根本不需要我orz。

神奇的Zip 🌈

题目描述

法外狂徒张三委托他的十佳员工卧底从某公司偷到了一份机密文件，这份文件是压缩过的，十佳员工告诉他文件没有加密，但张三把文件拿回家后发现压缩文件居然是加密的，他破口大骂，而此时十佳员工已经被送到非洲，无法为他解答困惑，于是张三找到了你，希望你能帮他获取到压缩包中的机密文件。在本任务中你需要尝试对task.zip压缩包进行解压才能获取到后续任务。
hints：它真的被加密了吗

题目分析

题目已经给出明示这是一个伪加密的zip文件。来了解一下zip文件的相关知识，一个 ZIP 文件由三个部分组成：压缩源文件数据区+压缩源文件目录区+压缩源文件目录结束标志，在WinHex中打开题目所给压缩包：

  • 压缩源文件数据区：

    项目值大小
    文件头标记504B03044 bytes
    解压文件所需pkware版本14002 bytes
    全局方式位标记（有无加密）02002 bytes
    ……

  • 压缩源文件目录区：

    项目值大小
    目录中文件文件头504B01024 bytes
    压缩使用的pkware版本14002 bytes
    解压所需的pkware版本14002 bytes
    全局方式位标记（有无加密）01002 bytes
    ……

  • 压缩源文件目录结束标志：

    项目值大小
    目录结束标记504B05064 bytes
    ……

WinHex搜索十六进制数504B0102查找到五个项，刚好是文件数目，将全局方式位标记全部改为0000成功解压。

题目总结

  • zip文件全局方式位标记为奇数时即可达到加密的效果。
  • 除了更改全局方式位标记使得压缩文件伪加密以外还可以更改文件头其他字段使得压缩文件“损坏”，如果遇到此种情况可以注意查看几个重点字段是否被更改。

Misc 📊

题目描述

法外狂徒张三似乎黔驴技穷了，为了找到第二个flag，他召唤出了他的道友：切·格瓦拉，切格瓦拉具有丰富的第三维度物体空间转移经验，他很快就帮你截获了一份秘密的网络流量(misc.pcapng)，切格瓦拉说道：“打工是不可能打工的，要不是看在我们多年搭档的份上我是不会轻易出手的！”，言罢转身帅气离去。现在flag就在流量包中，张三请求你把它解出！
可能用到的工具：

  • Wireshark —— 网络抓包/分析工具
  • WinHex —— 16进制编辑器
  • 在线PS —— 也许你会用到它

hints：

  • 你需要借助filter规则过滤掉无关的信息
  • 敲黑板：流量包里的flag并不是以文字形式存在哦

题目分析

从明示中不难看出本题flag在图片中，目的就是从流量包中找到该图片。扔进Wireshark一通分析，其实并不会过滤，只找到几张贴图库的图标，背景等无用的png，最后直接使用文件->导出对象->HTTP，马上发现了一张文件名全是数字的jpeg图片，保存打开是一张缺少定位符的二维码：

不能直接扫描，使用PS补全定位符扫描得到flag：We1come_to_D1an!

题目总结

  • 流量包中的一些文件通过HTTP传输，直接导出对象也许有意想不到的收获，更为快捷。
  • 图片可通过更改文件头等操作隐藏部分内容，本题虽然没有用到，但是需要注意。
  • 补全二维码定位符不需要太过精准的位置。

Reverse 🐛

题目描述

你和张三开始接触到网络安全很底层的一个方向——逆向工程，张三一看到满屏的汇编代码就傻眼了，开始萌生退意，但你作为新一代网络黑客、互联网杀手、QQ空间嘴炮王者、信息大类迷妹收割机，怎么会惧怕这亿点点困难？
我们已经为你准备了逆向工程的一些常用工具：

  • IDA Pro —— 反汇编利器，用于静态分析
  • OllyDbg —— PE文件动态调试工具
  • 汇编指令查询器 —— 助你迅速看懂x86汇编

(如果以下工具无法打开，建议关闭windows系统病毒和威胁防护设置中的实时保护，毕竟防止黑客的产生是防止病毒入侵的第一步)
我们建议你们先使用IDA进行静态分析，如果程序过于复杂，可以使用OllyDbg通过动态调试的方式直接
跳到最后一步！
无论遇到什么困难都不要怕，微笑着面对着它，消除恐惧的最好办法，那就是面对恐惧，坚持才是胜
利，加油吧！奥力给！
hints：IDA和Olldbg都可以搜索字符串哦！ IDA还能汇编转C哦！

终极任务 🍺

题目描述

恭喜你完成了上述众多的子任务，相信你已经对网络安全的CTF比赛有了一个大致的了解，那么，在终极任务中你将见识到网络安全的魅力所在，快去查看压缩包中的final.txt，了解终极任务的详细内容吧！

题目分析

将前面获得的三个flag进行异或得到final.zip的密码解压后得到final.txt：

恭喜你进入终极任务，想要拿到下一阶段的线索你需要去访问下列主机
IP：47.107.45.161
PORT：3002
该主机上有一个共享文件夹/home/myshare
使用smbclient即可访问
加油啊！少年！

看起来需要在Linux下访问共享文件夹，查越smbclient命令的使用方法后通过smbcilent -L 47.107.45.161 -p 3002查看你共享名，get得到hints.txt：

get shell by CVE-2017-7494 (Samba Vulnerability)
and then try to find ‘flag’
YOU ARE NOT ALLOWED TO DESTROY THE FLAG!!!
YOU ARE NOT ALLOWED TO DESTROY ANYTHING!!!

查找发现是一个漏洞，安装kali虚拟机后在mfs中并没找到该模块，只好到github上复制了一份，运行后设置好参数很容易就得到了shell：

一波find / -name "flag*"找到一堆看起来不怎么对的文件，随意翻了翻几个可能的文件夹也没有找到，考虑到flag应该是txt文件，查找所有txt文件：find / -name "*.txt"，于是找到了/root/.flag.txt，cat查看内容如下：

Congratulations ! you find the flag and win the game

Now you can write down your name and the time you find me

Rank name time

已经有两个人写入了自己的成绩，用echo "****" >> /root/.flag.txt将成绩写入末尾，完成final任务。
